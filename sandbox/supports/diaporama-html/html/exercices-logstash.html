Exercices Logstash

Accueil Hub exos

Exercices separes pour la partie ingestion/parsing Logstash.

1) Generer des logs de test

Creer un fichier logs avec INFO/WARN/ERROR.

Voir la solution
cat > logs/app.log <<'LOG'
2026-03-30T10:15:00Z INFO api - User created order
2026-03-30T10:16:45Z WARN billing - Payment retry
2026-03-30T10:17:12Z ERROR worker - Timeout
LOG

2) Lire le fichier avec input file

Configurer path/start_position/sincedb_path.

Voir la solution
input {
  file {
    path => "/usr/share/logstash/logs/*.log"
    start_position => "beginning"
    sincedb_path => "/dev/null"
  }
}

3) Parser avec grok

Extraire timestamp, level, service, log_message.

Voir la solution
filter {
  grok {
    match => {
      "message" => "%{TIMESTAMP_ISO8601:log_timestamp} %{LOGLEVEL:level} %{DATA:service} - %{GREEDYDATA:log_message}"
    }
    tag_on_failure => ["_grokparsefailure_app"]
  }
}

4) Normaliser la date

Mapper log_timestamp vers @timestamp.

Voir la solution
date {
  match => ["log_timestamp", "ISO8601"]
}

5) Sortir vers Elasticsearch

Indexer dans app-logs-YYYY.MM.dd.

Voir la solution
output {
  elasticsearch {
    hosts => ["http://elasticsearch:9200"]
    index => "app-logs-%{+YYYY.MM.dd}"
  }
}

6) Verifier l ingestion

Utiliser _cat/indices, _count, _search.

Voir la solution
curl "http://localhost:9200/_cat/indices?v"
curl "http://localhost:9200/app-logs-*/_count?pretty"
curl "http://localhost:9200/app-logs-*/_search?pretty"

7) Isoler les erreurs par service

Query filter level=ERROR + agg terms sur service.keyword.

Voir la solution
GET app-logs-*/_search
{
  "query": { "term": { "level": "ERROR" } },
  "aggs": {
    "by_service": { "terms": { "field": "service.keyword" } }
  }
}

8) Observer Logstash runtime

Verifier que le pipeline tourne sans erreur.

Voir la solution
curl "http://localhost:9600"
docker compose logs -f logstash
