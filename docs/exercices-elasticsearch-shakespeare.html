20 Exercices Elasticsearch Shakespeare

Accueil Hub exos

Objectif: pratiquer la logique de requete Elasticsearch sur le dataset Shakespeare en mode full-text + analytique. Chaque correction est masquee (clic pour afficher).

Bloc 1 - Setup et verification

1) Creer l index shakespeare

Mapping avec line_id, play_name, speech_number, line_number, speaker, text_entry.

Voir la solution
curl -X PUT "http://localhost:9200/shakespeare" -H "Content-Type: application/json" -d '{
  "mappings": {
    "properties": {
      "line_id": { "type": "integer" },
      "play_name": { "type": "keyword" },
      "speech_number": { "type": "integer" },
      "line_number": { "type": "keyword" },
      "speaker": { "type": "keyword" },
      "text_entry": { "type": "text" }
    }
  }
}'

2) Importer le dataset full

Utiliser le fichier JSON propose (ou l echantillon local en fallback).

Voir la solution
curl -L "https://raw.githubusercontent.com/grokify/kibana-tutorial-go/refs/heads/master/shakespeare.json" -o sandbox/data/shakespeare.json

curl -X POST "http://localhost:9200/shakespeare/_bulk?pretty" \
  -H "Content-Type: application/x-ndjson" \
  --data-binary @sandbox/data/shakespeare.json

# fallback rapide:
# curl -X POST "http://localhost:9200/shakespeare/_bulk" -H "Content-Type: application/x-ndjson" --data-binary @sandbox/data/shakespeare_sample.ndjson

3) Rendre les docs visibles et compter

Executer _refresh puis _count.

Voir la solution
curl -X POST "http://localhost:9200/shakespeare/_refresh"
curl "http://localhost:9200/shakespeare/_count?pretty"

4) Verifier le mapping reel

Afficher le mapping de l index.

Voir la solution
curl "http://localhost:9200/shakespeare/_mapping?pretty"

Bloc 2 - Requetes full-text et filtrage

5) Chercher les lignes contenant l idee de question

Voir la solution
GET shakespeare/_search
{
  "query": { "match": { "text_entry": "question" } }
}

6) Rechercher la phrase exacte to be, or not to be

Voir la solution
GET shakespeare/_search
{
  "query": {
    "match_phrase": {
      "text_entry": "to be or not to be"
    }
  }
}

7) Tolerance aux fautes sur queston

Voir la solution
GET shakespeare/_search
{
  "query": {
    "match": {
      "text_entry": {
        "query": "queston",
        "fuzziness": "AUTO"
      }
    }
  }
}

8) Lignes de Macbeth parlant de sleep

Voir la solution
GET shakespeare/_search
{
  "query": {
    "bool": {
      "must": [{ "match": { "text_entry": "sleep" } }],
      "filter": [{ "term": { "play_name": "Macbeth" } }]
    }
  }
}

9) Filtrer les lignes du speaker HAMLET

Voir la solution
GET shakespeare/_search
{
  "query": {
    "term": { "speaker": "HAMLET" }
  }
}

10) Trier les lignes de Hamlet par speech_number decroissant

Voir la solution
GET shakespeare/_search
{
  "query": { "term": { "play_name": "Hamlet" } },
  "sort": [{ "speech_number": "desc" }]
}

Bloc 3 - Aggregations metier

11) Compter les lignes par piece

Voir la solution
GET shakespeare/_search
{
  "size": 0,
  "aggs": {
    "by_play": { "terms": { "field": "play_name", "size": 20 } }
  }
}

12) Top 10 speakers par volume de lignes

Voir la solution
GET shakespeare/_search
{
  "size": 0,
  "aggs": {
    "top_speakers": { "terms": { "field": "speaker", "size": 10 } }
  }
}

13) Speech number moyen par piece

Voir la solution
GET shakespeare/_search
{
  "size": 0,
  "aggs": {
    "by_play": {
      "terms": { "field": "play_name", "size": 10 },
      "aggs": {
        "avg_speech": { "avg": { "field": "speech_number" } }
      }
    }
  }
}

14) Nombre de speakers distincts par piece

Voir la solution
GET shakespeare/_search
{
  "size": 0,
  "aggs": {
    "by_play": {
      "terms": { "field": "play_name", "size": 20 },
      "aggs": {
        "unique_speakers": { "cardinality": { "field": "speaker" } }
      }
    }
  }
}

15) Top 5 pieces contenant le terme love

Voir la solution
GET shakespeare/_search
{
  "size": 0,
  "query": { "match": { "text_entry": "love" } },
  "aggs": {
    "plays_with_love": {
      "terms": { "field": "play_name", "size": 5 }
    }
  }
}

Bloc 4 - Niveau avance

16) Highlight sur le terme blood

Voir la solution
GET shakespeare/_search
{
  "query": { "match": { "text_entry": "blood" } },
  "highlight": {
    "fields": {
      "text_entry": {}
    }
  }
}

17) Requete booleenne avec exclusion

Voir la solution
GET shakespeare/_search
{
  "query": {
    "bool": {
      "must": [{ "match": { "text_entry": "night" } }],
      "must_not": [{ "term": { "play_name": "A Midsummer Nights Dream" } }]
    }
  }
}

18) Pagination scalable avec search_after

Voir la solution
# page 1
GET shakespeare/_search
{
  "size": 3,
  "sort": [{"speech_number":"asc"}, {"line_id":"asc"}],
  "query": {"match_all": {}}
}

# page 2 (exemple)
GET shakespeare/_search
{
  "size": 3,
  "sort": [{"speech_number":"asc"}, {"line_id":"asc"}],
  "search_after": [1, 45],
  "query": {"match_all": {}}
}

19) Runtime field line_length + moyenne par piece

Voir la solution
GET shakespeare/_search
{
  "size": 0,
  "runtime_mappings": {
    "line_length": {
      "type": "long",
      "script": "emit(params._source.text_entry != null ? params._source.text_entry.length() : 0)"
    }
  },
  "aggs": {
    "by_play": {
      "terms": { "field": "play_name", "size": 10 },
      "aggs": {
        "avg_line_length": { "avg": { "field": "line_length" } }
      }
    }
  }
}

20) significant_text sur les lignes de Macbeth

Voir la solution
GET shakespeare/_search
{
  "size": 0,
  "query": { "term": { "play_name": "Macbeth" } },
  "aggs": {
    "key_terms": {
      "significant_text": {
        "field": "text_entry"
      }
    }
  }
}
