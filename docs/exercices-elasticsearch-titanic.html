20 Exercices Elasticsearch Titanic

Accueil Hub exos

Objectif: pratiquer la logique de requete Elasticsearch sur le dataset Titanic. Chaque correction est masquee (clic pour afficher).

Bloc 1 - Setup et verification

1) Creer l index titanic_data

Mapping avec survived, pclass, sex, age, fare, class, embark_town.

Voir la solution
curl -X PUT "http://localhost:9200/titanic_data" -H "Content-Type: application/json" -d '{
  "mappings": {
    "properties": {
      "survived": { "type": "integer" },
      "pclass": { "type": "integer" },
      "sex": { "type": "keyword" },
      "age": { "type": "float" },
      "fare": { "type": "float" },
      "class": { "type": "keyword" },
      "embark_town": { "type": "keyword" }
    }
  }
}'

2) Inserer un echantillon de 6 docs

Utiliser _bulk NDJSON.

Voir la solution
curl -X POST "http://localhost:9200/titanic_data/_bulk" -H "Content-Type: application/x-ndjson" -d '
{"index":{"_id":"1"}}
{"survived":0,"pclass":3,"sex":"male","age":22,"fare":7.25,"class":"Third","embark_town":"Southampton"}
{"index":{"_id":"2"}}
{"survived":1,"pclass":1,"sex":"female","age":38,"fare":71.2833,"class":"First","embark_town":"Cherbourg"}
{"index":{"_id":"3"}}
{"survived":1,"pclass":3,"sex":"female","age":26,"fare":7.925,"class":"Third","embark_town":"Southampton"}
{"index":{"_id":"4"}}
{"survived":1,"pclass":1,"sex":"female","age":35,"fare":53.1,"class":"First","embark_town":"Southampton"}
{"index":{"_id":"5"}}
{"survived":0,"pclass":3,"sex":"male","age":35,"fare":8.05,"class":"Third","embark_town":"Southampton"}
{"index":{"_id":"6"}}
{"survived":0,"pclass":2,"sex":"male","age":54,"fare":51.8625,"class":"Second","embark_town":"Southampton"}
'

3) Rendre les docs visibles et compter

Executer _refresh puis _count.

Voir la solution
curl -X POST "http://localhost:9200/titanic_data/_refresh"
curl "http://localhost:9200/titanic_data/_count?pretty"

4) Verifier le mapping reel

Afficher le mapping de l index.

Voir la solution
curl "http://localhost:9200/titanic_data/_mapping?pretty"

Bloc 2 - Requetes de filtrage

5) Lister tous les docs tries par fare desc

Voir la solution
curl -X GET "http://localhost:9200/titanic_data/_search?pretty" -H "Content-Type: application/json" -d '{
  "sort": [{"fare": "desc"}],
  "query": {"match_all": {}}
}'

6) Recuperer uniquement les femmes

Voir la solution
curl -X GET "http://localhost:9200/titanic_data/_search?pretty" -H "Content-Type: application/json" -d '{
  "query": { "term": { "sex": "female" } }
}'

7) Femmes de moins de 30 ans

Voir la solution
curl -X GET "http://localhost:9200/titanic_data/_search?pretty" -H "Content-Type: application/json" -d '{
  "query": {
    "bool": {
      "filter": [
        { "term": { "sex": "female" } },
        { "range": { "age": { "lt": 30 } } }
      ]
    }
  }
}'

8) Passagers de First ou Second class

Voir la solution
curl -X GET "http://localhost:9200/titanic_data/_search?pretty" -H "Content-Type: application/json" -d '{
  "query": { "terms": { "class": ["First", "Second"] } }
}'

9) Exclure les passagers de Third class

Voir la solution
curl -X GET "http://localhost:9200/titanic_data/_search?pretty" -H "Content-Type: application/json" -d '{
  "query": {
    "bool": {
      "must_not": [
        { "term": { "class": "Third" } }
      ]
    }
  }
}'

10) Passagers avec fare entre 10 et 60

Voir la solution
curl -X GET "http://localhost:9200/titanic_data/_search?pretty" -H "Content-Type: application/json" -d '{
  "query": {
    "range": { "fare": { "gte": 10, "lte": 60 } }
  }
}'

Bloc 3 - Aggregations metier

11) Calculer le taux de survie global

Voir la solution
GET titanic_data/_search
{
  "size": 0,
  "aggs": {
    "survival_rate": { "avg": { "field": "survived" } }
  }
}

12) Compter les passagers par sexe

Voir la solution
GET titanic_data/_search
{
  "size": 0,
  "aggs": {
    "by_sex": { "terms": { "field": "sex" } }
  }
}

13) Taux de survie par classe

Voir la solution
GET titanic_data/_search
{
  "size": 0,
  "aggs": {
    "by_class": {
      "terms": { "field": "class" },
      "aggs": {
        "survival_rate": { "avg": { "field": "survived" } }
      }
    }
  }
}

14) Age moyen par sexe

Voir la solution
GET titanic_data/_search
{
  "size": 0,
  "aggs": {
    "by_sex": {
      "terms": { "field": "sex" },
      "aggs": {
        "avg_age": { "avg": { "field": "age" } }
      }
    }
  }
}

15) Histogramme des fares (pas 10)

Voir la solution
GET titanic_data/_search
{
  "size": 0,
  "aggs": {
    "fare_histogram": {
      "histogram": { "field": "fare", "interval": 10 }
    }
  }
}

Bloc 4 - Niveau avance

16) Top 2 classes par nombre de passagers

Voir la solution
GET titanic_data/_search
{
  "size": 0,
  "aggs": {
    "by_class": {
      "terms": {
        "field": "class",
        "size": 2,
        "order": { "_count": "desc" }
      }
    }
  }
}

17) Plus age par classe (top_hits)

Voir la solution
GET titanic_data/_search
{
  "size": 0,
  "aggs": {
    "by_class": {
      "terms": { "field": "class" },
      "aggs": {
        "oldest_passenger": {
          "top_hits": {
            "sort": [{ "age": { "order": "desc" } }],
            "size": 1,
            "_source": ["sex", "age", "fare", "survived"]
          }
        }
      }
    }
  }
}

18) Percentiles de fare (50, 90, 99)

Voir la solution
GET titanic_data/_search
{
  "size": 0,
  "aggs": {
    "fare_percentiles": {
      "percentiles": {
        "field": "fare",
        "percents": [50, 90, 99]
      }
    }
  }
}

19) Pagination scalable avec search_after

Page 1 puis page suivante avec le sort du dernier hit.

Voir la solution
# page 1
GET titanic_data/_search
{
  "size": 2,
  "sort": [{"fare":"asc"}, {"_id":"asc"}],
  "query": {"match_all": {}}
}

# page 2 (exemple)
GET titanic_data/_search
{
  "size": 2,
  "sort": [{"fare":"asc"}, {"_id":"asc"}],
  "search_after": [7.25, "1"],
  "query": {"match_all": {}}
}

20) Runtime field is_child + aggregation

Voir la solution
GET titanic_data/_search
{
  "size": 0,
  "runtime_mappings": {
    "is_child": {
      "type": "keyword",
      "script": "emit(doc['age'].size()!=0 && doc['age'].value < 18 ? 'yes' : 'no')"
    }
  },
  "aggs": {
    "children_split": {
      "terms": { "field": "is_child" }
    }
  }
}
