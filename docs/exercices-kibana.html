Exercices Kibana

Accueil Hub exos

Objectif: passer d'une exploration Discover a une analyse dashboard complete, en validant techniquement chaque visuel.

1) Creer le Data View Shakespeare

Objectif: connecter Kibana a l'index shakespeare sans erreur de pattern.

Resultat attendu: Data View cree et visible dans la liste Stack Management.

Voir la solution
Stack Management -> Data Views -> Create data view
Name: Shakespeare
Index pattern: shakespeare
Time field: none
Save

2) Ouvrir Discover et configurer un affichage utile

Objectif: lire rapidement la structure des documents via les bonnes colonnes.

Resultat attendu: tableau avec play_name, speaker, speech_number, text_entry.

Voir la solution
Discover -> choisir Data View Shakespeare
Ajouter les champs en colonnes
Trier par speech_number desc
Sauvegarder la recherche: "Shakespeare QA"

3) Ecrire un filtre KQL pour HAMLET

Objectif: appliquer un filtre exact sur un champ categoriel.

Resultat attendu: uniquement des lignes dont le speaker est HAMLET.

Voir la solution
speaker: "HAMLET"

4) Ecrire un filtre KQL combine (piece + terme)

Objectif: croiser contrainte de contexte et critere textuel.

Resultat attendu: lignes de Macbeth contenant le terme sleep.

Voir la solution
play_name: "Macbeth" and text_entry: "sleep"

5) Creer un visuel Lens "Lines by play"

Objectif: mesurer la repartition du corpus par piece.

Resultat attendu: graphique en barres avec play_name en axe X et Count en metrique.

Voir la solution
Visualize Library -> Create visualization -> Lens
Drag play_name sur X
Metric: Count
Save: "Lines by play"

6) Creer un visuel Lens "Top speakers"

Objectif: identifier les personnages les plus presents.

Resultat attendu: top 10 speakers tries en decroissant.

Voir la solution
Create Lens chart
Axis: speaker
Metric: Count
Sort: Desc
Size: 10
Save: "Top speakers"

7) Ecrire une formule Lens pour la part de Macbeth

Objectif: manipuler une proportion metier simple dans Kibana.

Resultat attendu: une valeur comprise entre 0 et 1 (ou pourcentage).

Voir la solution
count(kql='play_name: "Macbeth"') / count()

8) Construire un dashboard de synthese

Objectif: assembler plusieurs visuels dans une vue coherente.

Resultat attendu: dashboard sauvegarde avec au moins 3 panels.

Voir la solution
Create dashboard
Add panels:
- Lines by play
- Top speakers
- Speech number distribution
Save dashboard: "Shakespeare Overview"

9) Ajouter des controls interactifs de filtrage

Objectif: permettre une exploration sans ecrire de KQL.

Resultat attendu: options list fonctionnelles sur play_name et speaker.

Voir la solution
Dashboard -> Controls
Add options list: play_name
Add options list: speaker
Apply to all panels

10) Inspecter la requete reelle d'un panel

Objectif: verifier que Kibana envoie bien le DSL attendu.

Resultat attendu: requete JSON copiable depuis Inspect -> Requests.

Voir la solution
Sur un panel -> menu (...) -> Inspect -> Requests
Copier la requete
Coller dans Dev Tools pour verification

11) Rejouer en Dev Tools l aggregation "by_play"

Objectif: comparer UI et resultat DSL brut.

Resultat attendu: les buckets Dev Tools correspondent aux valeurs du panel Lens.

Voir la solution
GET shakespeare/_search
{
  "size": 0,
  "aggs": {
    "by_play": {
      "terms": { "field": "play_name", "size": 20 }
    }
  }
}

12) Exporter les objets Kibana du TP

Objectif: conserver et partager le travail (data view + visuels + dashboard).

Resultat attendu: fichier .ndjson exporte depuis Saved Objects.

Voir la solution
Stack Management -> Saved Objects -> Export
Selectionner:
- Data view Shakespeare
- Visualisations Lens creees
- Dashboard Shakespeare Overview
Exporter en .ndjson
