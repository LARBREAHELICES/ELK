TP 00 - Kibana pas à pas

Retour Exercices

Dataset shakespeare - parcours guidé pour construire un dashboard lisible.

Objectif

  1. Créer une visualisation Lens : répliques par pièce
  2. Créer une visualisation Lens : top speakers
  3. Créer une visualisation Lens : moyenne des années
  4. Assembler un dashboard final

Étape 0 - Vérifier le prérequis (index avant Kibana)

Important : Kibana ne crée pas l'index Elasticsearch. Il crée un Data View qui pointe vers un index existant.

Ordre correct :

  1. index Elasticsearch existant (shakespeare)
  2. Data View Kibana (shakespeare)
  3. visualisations Lens

Contrôle rapide de l'index

GET _cat/indices/shakespeare?v
GET shakespeare/_count

Étape 1 - Commencer par le champ datetime

Pourquoi c'est important :

  • Kibana utilise un champ date pour les graphiques temporels
  • Le filtre de temps global dépend de ce champ
  • Sans date, certaines visualisations semblent vides

1.1 Créer le champ year_date

PUT shakespeare/_mapping
{
  "properties": {
    "year_date": { "type": "date" }
  }
}

1.2 Alimenter year_date depuis year

POST shakespeare/_update_by_query
{
  "script": {
    "source": """
      if (ctx._source.year != null) {
        ctx._source.year_date = ctx._source.year + '-01-01T00:00:00Z';
      }
    """
  }
}

1.3 Contrôle rapide

GET shakespeare/_search
{
  "size": 1,
  "_source": ["year", "year_date"]
}

Étape 2 - Créer le Data View

Chemin : Stack Management -> Data Views

  • Name : shakespeare
  • Index pattern : shakespeare
  • Time field : year_date (recommandé)

Si vous ne faites pas de timeline, vous pouvez Skip time field.

Étape 3 - Ouvrir Visualize Library

Analytics -> Visualize Library -> Create new visualization

Étape 4 - Choisir Lens et comprendre les autres choix

Pour ce TP, choisir Lens :

  • éditeur drag-and-drop rapide
  • idéal pour démarrer et prototyper

Les autres choix de la fenêtre

  • Maps : cartes géographiques
  • TSVB : séries temporelles avancées
  • Custom visualization : Vega, visuels très personnalisés
  • Aggregation based : ancien éditeur orienté agrégations
  • Text : bloc texte dans un dashboard

Étape 5 - Lens #1 : répliques par pièce

  • Type : Bar vertical
  • Horizontal axis : play_name
  • Vertical axis : Count of records

Étape 6 - Lens #2 : top 5 speakers

  • Horizontal axis : speaker
  • Vertical axis : Count
  • Tri : Descending
  • Limite : 5

Étape 7 - Lens #3 : moyenne des années

  • Horizontal axis : play_name
  • Vertical axis : Average(year)

Count = volume, Average = métrique calculée.

Étape 8 (option) - Timeline dans Lens

  • Type : Line
  • X-axis : Date histogram(year_date)
  • Y-axis : Count

Si le graphique est vide : élargir le time range et vérifier le Data View.

Étape 9 - Dashboard final

Dans Dashboard -> Create, ajouter :

  • Répliques par pièce
  • Top speakers
  • Moyenne des années

Checklist de rendu

  1. Champ year_date créé et alimenté
  2. Data View shakespeare prêt
  3. 3 visualisations Lens minimum
  4. 1 dashboard final
  5. Explication des axes : horizontal = regroupement, vertical = calcul
